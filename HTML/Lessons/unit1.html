Java Basics

            

                Java is like a magic wand that helps you create cool apps and software for your phone and computer! Lots of awesome apps like Minecraft and Netflix are made with Java. When we write a program in Java, we write it inside something called a "class", which is like a blueprint for making things.

The main method is like the boss of the class, and it tells the program what to do. You can change the code inside the main method to make the program do different things. Like, you could make it say "Hello, [your name]!" instead of just "Hello, world!"
Check out the following code below to learn the syntax for making a class and a main method. Just a note, the built in ide will only recognize a class name of "Main".

    
public class Main {
    public static void main(String[] args) {
        //Write your code here!!!
    }
}
    




To make the program talk, we use something called "print statements". It's like telling the program to speak. There are two types: System.out.println() and System.out.print(). System.out.println() talks and goes to a new line, while System.out.print() talks without going to a new line. 
We use double quotes to make the computer understand that we're making it say something. Text that is in double quotes like "Hello, world!" are called String Literals. 
And just like in English, we end each command with a semicolon (;).

But wait! Sometimes our code has mistakes, and that's okay! It just means we have a bug in our program. The reason we have these bugs is because the computer is trying to translate our code into machine code through a process called "compiling". The computer needs the translation in order to do understand our instructions and do stuff, but if we use incorrect syntax, the compiler won't be able to translate correctly. In order to fix these bugs, we must do something called "debugging". Even fancy coders have bugs, so don't worry if you do too! Sometimes it helps to explain our code to a rubber duck or ask a friend for help.

We can also leave notes for ourselves or other coders using "comments". These are like little secrets hidden inside our code that the computer ignores. 
We can use them to explain what our code does or why we wrote it a certain way. Just like in a book, we use different types of comments for different things.
To write them, use // for single line comments and /* and */ for multi-line comments. The following snippet of code below has a bug in it, can you figure out why?
            

            
                
public class Main {
   public static void main(String[] args) {
        System.out.println("Hello world!); //Try to figure out why this is causing a bug
    }
}
                
            

        

        

            
Variables and Data Types

            

                Alright, let's talk about variables and primitive data types!

So, imagine you have a box with your name on it. You can put anything you want in it, like a toy or a snack. Well, a variable is like that box - but for the computer. It's a container that can hold different things, like numbers or words.

There are two types of variables: primitive and object/reference. Primitive variables are for basic stuff like whole numbers (int), decimal numbers (double), and true/false values (boolean). String is an object type that holds a bunch of letters.

When you make a variable, you need to give it a name and tell the computer what type of stuff it's going to hold. Under the hood, the computer allocates a location in the memory to store the data. That location holds whatever value you put in it. Whenever you want to use the variable later, you just call it by its name. 
You can even change what's inside the box by assigning a new value to the variable!
But wait, there are some rules for naming variables. You can't use words that Java already knows, like "if" or "int". Also keep in mind that if you use the keyword "final" in a variable declaration, the variable can not be reassigned a different value later. Remember to make your names descriptive, like "score" or "playerName". 
Also, use lowercase letters to start the name and capitalize the first letter of each new word. This is called Camel Case and it's structured like this: camelCase.

So, there you have it - variables are like magic boxes for the computer that can hold different kinds of stuff. Just remember to name them wisely! Check out the following code to see the syntax of how to declare, initialize, and assign values to variables.
            


    
public class Main {
    public static void main(String[] args) {
        final double gpa = 3.5;
        boolean pass = true;
        int grade = 97;
        System.out.print(gpa); //Prints 3.5
    }
}
    


        

        

            
Arithmetic Operators

            

                I hope you like math cause we're gonna use it in our code. If you paid attention, you'll remember the variable-box analogy. 
                In certain programs, we can use variables to store input values given by users, but that's outside the scope of this website.

To assign a value to a variable, we use the = sign. It's like saying "this variable gets this value." And when we want to copy a variable's value to another variable without changing the original variable, we can do that too!

In games, we can use variables to keep score. And when we want to add one to the score, we use a funny operation called "incrementing." It looks like this: score = score + 1. It might look weird, but it means the score variable gets its old value plus one.

In Java, we can use math symbols like +, -, *, and / to do math. If we use them with whole numbers, we'll get another whole number as the answer. But if we use them with decimals, we'll get a decimal number as the answer.

To check if two things are the same, we use ==. To check if they're not the same, we use !=. Be careful not to mix up = with ==. And when we want to make sure things happen in the right order, we can use parentheses (), like in PEMDAS (or BODMAS if you're bri'ish).

Finally, the % sign is called the mod operator. It tells you the remainder you get when you divide one number by another number. For example, 5 % 2 is 1 because 2 goes into 5 two times with a remainder of 1. And if the first number is smaller than the second number, the result will be the first number.
For example, 2 % 3 will equal 2. Also, DON'T DIVIDE BY 0! You'll get an error, obviously.
So go ahead and have some fun with math and variables in your code! Below is some sample code you can try out for yourself.
            


    
public class Main {
    public static void main(String[] args) {
        int score = 3;
        score = score + 2; //adds 2 to score
        score = score % 2; //modulos score by 2
        System.out.println(score); //Prints 1
        double score2 = 1.0;
        System.out.println(score / 2); //Prints 0 because score is an integer and rounded down
        System.out.println(score2 / 2); //Prints 0.5 because score2 is a double
    }
}
    


        

        

            
Compound Assignment

            

                Are reassignment statements too much for you to type?  Well don't worry, cause compound assignment operators are here to help you!

These shortcuts can do a math operation and assignment in one step! 
For example, instead of writing x = x + 1 to add 1 to a number called x, you can write x += 1! And guess what? It works with other operators too like +, -, *, /, and %!

Now, let's talk about the coolest shortcut operators, ++ and --. The ++ adds 1 to a number and -- subtracts 1. You can't use them with other operators, but they're still pretty awesome! So, x++ means the same as x += 1 and x = x + 1. And yep, you guessed it, the -- does the opposite!

These shortcuts are like magic spells that help you save time and make your code shorter! So go ahead and use them! And remember, on the exam you can use either x++ or ++x to add 1 to a value. The only difference is when you're assigning it to another variable. So go forth and conquer the coding world with your amazing new skills! The code below replicates the previous code, but with compound assignment operators instead.
            


    
public class Main {
    public static void main(String[] args) {
        int score = 3;
        score += 2;
        score %= 2;
        System.out.println(score); //Prints 1
    }
}
    


        

        

            
Type Casting

            

                Type casting in Java is like changing the "shape" of a variable. It's like when you take a ball of clay and shape it into a cool mug or bowl. Type casting lets you change a variable's type from one to another. So, you can turn a number into a temporary value in a different data type using (int) or (double).

Now, when you divide two integers, Java assumes you want an integer result. But, if you use both integers and decimals, Java thinks you want a double result. And if you want to get a double result from a math operation with integers, you can use casting.

But wait, there's more! You can also round a double to the nearest integer using (int)(x + 0.5) or (int)(x - 0.5) for negative numbers. So, if you have a decimal number that's almost a whole number, you can use casting to round it to the nearest integer.

But remember, Java has a limit for how many digits it can store for any double number. That means calculations on computers might not always be accurate. And finally, int values in Java have a limited range from Integer.MIN_VALUE to Integer.MAX_VALUE. If an int value goes beyond that range, it could result in an incorrect value. So, it's important to keep track of your variables and make sure they don't go out of bounds!
            


    
public class Main {
    public static void main(String[] args) {
        int score = 3;
        System.out.println(score / 2); //Prints 1
        System.out.println((double)score / 2); //Prints 1.5
    }
}
